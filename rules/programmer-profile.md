# Programmer Profile and Experience Levels

## Experience Level Tags
The following tags should be used beside certain rules to denote experience level. When implementing or explaining parts of the stack, adjust explanations based on the experience level indicated:

- **Beginner** - Never implemented or used the tech
- **Novice** - Seen it, but can't use it well
- **Intermediate** - Implemented the work and generally understand what I'm reading
- **Proficient** - Regularly implemented the work and need little hand-holding
- **Expert** - Know what I'm doing

## Stack Experience Levels (for this project)
- Vue 3: **Proficient**
- Script Setup: **Proficient**
- Composition API: **Proficient**
- TypeScript: **Intermediate**
- Less: **Proficient**
- Pinia: **Novice**

**Note:**
- Always provide explanations and code comments at the level indicated by these tags.
- Reference these tags in other rules to clarify the assumed depth of explanation.
